# API Error & Silent Policy

## 目标

统一前端 API 调用时的错误提示行为，避免页面重复弹错、重复 toast、以及静默失败不可见。

## 调用规则

- 默认调用（`silent` 不传）：
  - 由 `services/request.ts` 统一弹错
  - 适用于用户主动触发的重要操作（提交、删除、状态切换）
- 静默调用（`silent: true`）：
  - 只在“后台刷新/并行请求/附属信息拉取”场景使用
  - 页面自行决定是否提示（如只在失败时显示一次轻提示）

## 推荐场景

- `silent = false`：
  - 登录、登出、创建/编辑/删除等主流程动作
- `silent = true`：
  - 页面初始化时的非核心附加信息（例如统计角标、配置分组、健康检查）

## 状态码约定

- `401`：请求层统一清理认证并触发未授权事件
- `403`：权限不足，建议页面跳转到 `/403`
- `4xx`：参数或业务错误，优先展示后端 `message`
- `5xx`：统一显示服务端错误提示

## 页面层建议

- 页面不要重复 try/catch 弹同类错误（避免双提示）
- 同一个操作链条只保留一次最终提示

## 1. 测试策略

### 1.1 测试框架

- **pytest** + **pytest-asyncio**（异步测试）
- **httpx** 测试客户端（FastAPI 测试）
- **工厂模式**：测试数据通过工厂函数生成

### 1.2 测试分层

| 层级 | 类型 | 覆盖范围 | 工具 |
|------|------|----------|------|
| 单元测试 | 函数/类级别 | 核心逻辑、数据校验、工具函数 | pytest |
| 集成测试 | 插件级别 | 插件路由 + 数据库交互 | pytest + httpx |
| 端到端测试 | 全链路 | 用户完整操作流程 | httpx 测试客户端 |

### 1.3 测试原则

- 测行为，不测实现细节
- 覆盖所有可预测的行为路径（输入 → 输出）
- 失败信息直接指出哪个行为路径有问题
- 按模块/工作流组织目录

## 2. 核心层测试

### 2.1 服务容器

| 测试 | 场景 |
|------|------|
| `test_register_and_get` | 注册服务 → 获取 → 返回正确实例 |
| `test_lazy_creation` | 注册后不获取 → 服务未被创建 |
| `test_circular_dependency` | A→B→A → 抛出 CircularDependencyError |
| `test_is_available` | 注册前返回 False，注册后返回 True |
| `test_shutdown` | shutdown 后再次 get → 抛出异常 |
| `test_missing_service` | get 不存在的 key → 抛出 KeyError |

### 2.2 配置管理

| 测试 | 场景 |
|------|------|
| `test_env_file_loading` | .env 中的值被正确加载 |
| `test_env_override` | 环境变量覆盖 .env 值 |
| `test_get_required_missing` | 缺少必须配置 → 启动失败 |
| `test_get_with_default` | get 不存在 key 返回默认值 |

### 2.3 插件发现

| 测试 | 场景 |
|------|------|
| `test_discover_plugins` | 扫描目录 → 正确 import 所有插件 |
| `test_skip_non_plugin_dirs` | 没有 `__init__.py` 的目录被跳过 |
| `test_plugin_registration` | 插件自注册 → registry 中有对应 key |

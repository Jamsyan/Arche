## 1. 本地开发环境

### 1.1 基础设施

```yaml
# docker-compose.dev.yml
services:
  postgres:
    image: postgres:17-alpine
    environment:
      POSTGRES_DB: veil
      POSTGRES_USER: veil
      POSTGRES_PASSWORD: veil
    ports:
      - "5432:5432"
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: pg_isready -U veil
      interval: 5s
      timeout: 3s
      retries: 5

volumes:
  pgdata:
```

### 1.2 开发启动流程

```bash
# 1. 启动 PostgreSQL
docker compose -f docker-compose.dev.yml up -d

# 2. 同步后端依赖
uv sync

# 3. 启动后端
uvicorn backend.main:app

# 4. 启动前端
cd frontend && npm install && npm run dev
```

### 1.3 环境变量

```bash
# .env（开发）
DATABASE_URL=postgresql+asyncpg://veil:veil@localhost:5432/veil
SECRET_KEY=dev-secret-key-change-in-prod

OSS_ACCESS_KEY_SECRET=
GITHUB_PAT=
CRAWLER_SEEDS=https://www.zhihu.com,https://tieba.baidu.com,...
```

## 2. 生产部署

### 2.1 基础设施

```yaml
# docker-compose.prod.yml
services:
  postgres:
    image: postgres:17-alpine
    environment:
      POSTGRES_DB: veil
      POSTGRES_USER: veil
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    ports:
      - "127.0.0.1:5432:5432"  # 仅本机可访问
    volumes:
      - pgdata:/var/lib/postgresql/data
      - ./backup:/backup       # 定期备份目录
    restart: unless-stopped

  backend:
    build:
      context: .
      dockerfile: Dockerfile.backend
    environment:
      DATABASE_URL: postgresql+asyncpg://veil:${DB_PASSWORD}@postgres:5432/veil
      SECRET_KEY: ${SECRET_KEY}
      STORAGE_PATH: /data/storage
    volumes:
      - storage_data:/data/storage  # 持久化存储
      - ./frontend/dist:/app/frontend/dist:ro  # 前端构建产物
    ports:
      - "127.0.0.1:8000:8000"
    depends_on:
      postgres:
        condition: service_healthy
    restart: unless-stopped

  streamlit:
    build:
      context: .
      dockerfile: Dockerfile.streamlit
    environment:
      DATABASE_URL: postgresql+asyncpg://veil:${DB_PASSWORD}@postgres:5432/veil
    ports:
      - "127.0.0.1:8501:8501"
    depends_on:
      - backend
    restart: unless-stopped

volumes:
  pgdata:
  storage_data:
```

### 2.2 Nginx 反代（生产）

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate     /etc/letsencrypt/live/your-domain/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain/privkey.pem;

    # 前端
    location / {
        proxy_pass http://127.0.0.1:8000;
    }

    # Streamlit 面板（仅 P0 可访问，加一层 Basic Auth）
    location /ops/ {
        auth_basic "Admin Only";
        auth_basic_user_file /etc/nginx/.htpasswd;
        proxy_pass http://127.0.0.1:8501;
    }
}
```

### 2.3 数据库备份

```bash
# 每日凌晨备份
0 2 * * * pg_dump -U veil veil > /backup/veil-$(date +\%Y\%m\%d).sql
# 保留 30 天
find /backup -name "*.sql" -mtime +30 -delete
```

### 2.4 后端 Dockerfile

```dockerfile
FROM python:3.12-slim

WORKDIR /app
COPY . .

RUN pip install uv
RUN uv sync --frozen

EXPOSE 8000

CMD ["uv", "run", "uvicorn", "backend.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### 2.5 Streamlit Dockerfile

```dockerfile
FROM python:3.12-slim

WORKDIR /app
COPY . .

RUN pip install uv
RUN uv sync --frozen --group streamlit

EXPOSE 8501

CMD ["uv", "run", "streamlit", "run", "backend/plugins/cloud/ui/app.py", "--server.port=8501"]
```

## 3. 部署检查清单

### 3.1 首次部署

- [ ] 安装 Docker + Docker Compose
- [ ] 生成 SECRET_KEY（`openssl rand -hex 32`）
- [ ] 配置 .env 文件
- [ ] 配置 PostgreSQL 密码
- [ ] 配置阿里云 OSS 密钥（可选）
- [ ] 配置 GitHub PAT（可选）
- [ ] 创建 P0 用户（首次注册用户默认 P0，或手动插入数据库）
- [ ] 构建前端（`cd frontend && npm run build`）
- [ ] `docker compose up -d`
- [ ] 验证 PostgreSQL 连接
- [ ] 验证后端 API（`curl http://localhost:8000/docs`）
- [ ] 验证前端访问

### 3.2 日常运维

- [ ] 检查磁盘使用量（40GB 限制）
- [ ] 检查数据库备份是否成功
- [ ] 检查 OSS 冷热分层是否正常运行
- [ ] 检查爬虫日志
- [ ] 检查 Nginx SSL 证书有效期

## 4. 存储规划

```
/data/storage/
├── users/            # 用户上传文件
├── p1/               # P1 独立空间（10GB 限额）
├── external/         # 外部写入
├── private/          # 个人私有文件
│   ├── hot/          # 热存储
│   └── cold/         # 已推阿里云的文件元数据
└── cache/            # GitHub 代理缓存
```

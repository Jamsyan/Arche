## 2. 核心层设计

### 2.1 目录结构

```
backend/
├── core/
│   ├── __init__.py          # create_app() 工厂
│   ├── base_plugin.py       # BasePlugin 抽象基类
│   ├── plugin_registry.py   # PluginRegistry 单例 + discover_plugins()
│   ├── container.py         # 服务容器（注册/发现/循环检测）
│   └── config.py            # 配置管理（env / config file）
├── plugins/                 # 插件目录（每个插件一个子目录）
│   ├── auth/
│   ├── blog/
│   ├── oss/
│   ├── github_proxy/
│   ├── asset_mgmt/
│   ├── crawler/
│   └── cloud/
└── main.py                  # 入口
```

### 2.2 服务容器

```python
class ServiceContainer:
    def register(self, name, factory)      # 注册服务工厂
    def get(self, name)                     # 惰性获取，追踪调用栈
    def is_available(self, name)            # 检查服务是否存在
```

- **惰性创建**：服务在第一次 `get()` 时才实例化
- **调用栈追踪**：维护 `_resolving` set，检测到循环立即拒绝
- **服务生命周期**：跟随 App 生命周期，App 关闭时统一清理

### 2.3 配置管理

```python
class Config:
    def get(key, default)     # 获取配置值
    def get_required(key)     # 获取必须存在的配置
```

- 从 `.env` 文件加载（python-dotenv）
- 配置分层：默认值 < .env < 环境变量
- 插件通过容器获取 `Config` 实例，不直接读 env

### 2.4 数据库集成

```python
# core/__init__.py
from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession

async def init_db(config: Config):
    engine = create_async_engine(config.get("DATABASE_URL"))
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
    return engine
```

- 异步 SQLAlchemy（`asyncpg` 驱动）
- Base.metadata 统一注册，插件可贡献自己的 model
- 连接池：核心创建，插件通过容器获取 `AsyncSession`

## 3. 插件层设计

### 3.1 插件接口

```python
class BasePlugin(ABC):
    name: str = ""
    version: str = "0.1.0"
    requires: list[str] = []      # 必须有，没有就启动失败
    optional: list[str] = []      # 没有就优雅降级

    @abstractmethod
    def setup(self, app: FastAPI) -> None:
        """注册路由、中间件、服务"""

    def register_services(self, container: ServiceContainer) -> None:
        """可选：注册服务到容器。默认空实现"""
        pass

    def on_startup(self) -> None:
        """可选：启动后钩子。默认空实现"""
        pass

    def on_shutdown(self) -> None:
        """可选：关闭前钩子。默认空实现"""
        pass
```

### 3.2 插件标准目录

```
backend/plugins/{plugin_name}/
├── __init__.py      # 入口：插件类 + 自注册
├── routes.py        # FastAPI 路由
├── services.py      # 业务逻辑
├── models.py        # SQLAlchemy 模型
└── ...              # 按需拆分
```

- 简单插件可只放 `__init__.py`
- 复杂插件按 `routes.py` / `services.py` / `models.py` 分层

### 3.3 插件发现与激活流程

```
阶段1: discover_plugins() → 扫描 plugins/ 下所有子目录 → import __init__.py
阶段2: 每个 __init__.py 调用 registry.register(name, plugin)
阶段3: DAG 拓扑排序（基于 requires/optional 声明）
阶段4: create_app() → registry.activate_all(app) 按顺序 setup
阶段5: 各插件调用 register_services() 注册服务到容器
阶段6: resolve 服务依赖（此时所有服务都已注册）
阶段7: on_startup() 钩子
```

## 4. 前端设计

### 4.1 目录结构

```
frontend/src/
├── App.vue                     # 根据等级切换 shell
├── main.js                     # Vue 入口
├── router/
│   ├── index.js               # Vue Router
│   ├── component-registry.js   # 等级→组件映射
│   └── auth.js                # useAuth()
└── components/
    ├── blog/                   # 公开（等级 4 可见）
    ├── platform/               # 登录（等级 3+ 可见）
    ├── admin/                  # 管理（等级 0 可见）
    └── ops/                    # 运维面板（等级 0 可见）
```

### 4.2 按需下发机制

1. Vite 自动按 dynamic import 分割 chunk
2. 登录后后端返回用户等级
3. 前端根据等级从 `component-registry.js` 获取可用路由
4. 只 import() 对应等级的组件
5. 高等级组件 URL 不写到低等级页面

## 5. 关键流程

### 5.1 用户认证流程

```
浏览器 → POST /api/auth/login → auth 插件验证 → 返回 token + 等级
浏览器 → GET /api/xxx + Bearer token → auth 中间件校验等级 → 放行/拒绝
```

### 5.2 文件上传流程（用户）

```
用户 → POST /api/oss/upload + token → oss 插件
    → 验证 token 和等级
    → 写入本地存储（/storage/user/{user_id}/）
    → 返回文件 ID/URL
```

### 5.3 外部写入流程（其他项目）

```
外部客户端 → POST /api/oss/external/{tenant_id}/upload
    → 验证 AK/SK（不能越界访问其他租户）
    → 路径安全检查（禁止 ../）
    → 写入 /storage/external/{tenant_id}/
```

### 5.4 云训练流程

```
用户(等级0) → POST /api/cloud/tasks → 创建任务
    → orchestrator: 创建云实例(智云星)
    → SSH 部署代码(git clone + uv sync)
    → 启动训练(tmux)
    → 采集 GPU 指标(pynvml)
    → 训练完成 → SFTP 回传产物
    → SHA256 校验 → 暂存
    → 销毁实例
```

## 7. 安全设计

### 7.1 API 安全

- 所有 API 请求必须携带 Bearer token
- 等级校验中间件：`@require_level(1)` 装饰器
- 跨域：仅允许前端 dev server 和 prod domain

### 7.2 文件安全

- 路径隔离：`/storage/{type}/{tenant_id}/` 不可越界
- 禁止 `../`、禁止绝对路径
- 上传文件类型白名单
- 文件大小限制

### 7.3 爬虫合法性

- 遵守 robots.txt
- 模拟浏览器 UA
- 限速（同站请求间隔 ≥ N 秒）
- 不爬取敏感内容

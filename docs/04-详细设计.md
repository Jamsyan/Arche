## 1. 核心层详细设计

### 1.1 create_app() 工厂


### 1.3 服务容器（ServiceContainer）

```python
# backend/core/container.py
class ServiceContainer:
    def __init__(self):
        self._factories: dict[str, Callable] = {}
        self._instances: dict[str, object] = {}
        self._resolving: set[str] = set()  # 调用栈追踪

    def register(self, name: str, factory: Callable) -> None:
        """注册服务工厂函数"""
        self._factories[name] = factory

    def get(self, name: str) -> object:
        """惰性获取，循环检测"""
        if name in self._resolving:
            raise CircularDependencyError(f"循环依赖: {' -> '.join(self._resolving)} -> {name}")
        if name not in self._instances:
            self._resolving.add(name)
            try:
                self._instances[name] = self._factories[name](self)
            finally:
                self._resolving.discard(name)
        return self._instances[name]

    def is_available(self, name: str) -> bool:
        return name in self._factories

    def shutdown(self) -> None:
        """统一关闭所有服务"""
        for instance in reversed(self._instances.values()):
            if hasattr(instance, "close"):
                instance.close()
```

**内置服务**：
- `config` — Config 实例
- `db` — 数据库引擎 + Session 工厂
- `auth` — AuthService（由 auth 插件注册）
- `storage` — StorageService（由 oss 插件注册）

### 1.4 数据库集成

```python
# backend/core/db.py
from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession, async_sessionmaker
from sqlalchemy.orm import DeclarativeBase

Base = DeclarativeBase()

async def init_db(database_url: str):
    engine = create_async_engine(database_url, echo=False)
    session_factory = async_sessionmaker(engine, class_=AsyncSession)

    async with engine.begin() as conn:
        # 插件在 __init__.py 中 import models.py，
        # 所以 create_all 前所有 Model 都已注册到 Base
        await conn.run_sync(Base.metadata.create_all)

    return engine, session_factory
```

**插件贡献 Model**：
插件在 `__init__.py` 中 import 自己的 `models.py`，SQLAlchemy Model 继承自 `core.db.Base`，`create_all` 时自动建表。

## 2. 插件详细设计

### 2.1 auth 插件

**路由**：
- `POST /api/auth/register` — 注册（默认 P5）
- `POST /api/auth/login` — 登录
- `POST /api/auth/logout` — 登出
- `GET /api/auth/me` — 获取当前用户信息
- `POST /api/auth/refresh` — 刷新 token

**服务**：
- `AuthService` — 用户 CRUD、密码验证、JWT 签发/验证、等级查询

**中间件**：
- `require_level(n)` — 装饰器，检查用户等级 ≤ n（数字越小权限越高）

**数据模型**：
```
users: id, username, password_hash, level, blog_quality_level, created_at, updated_at
```

### 2.2 blog 插件

**路由**：
- `GET /api/blog/posts` — 帖子列表（公开，按质量等级/浏览量排序）
- `GET /api/blog/posts/{slug}` — 帖子详情（公开）
- `POST /api/blog/posts` — 发帖（需登录，进入审核队列）
- `PUT /api/blog/posts/{id}` — 编辑（作者本人）
- `DELETE /api/blog/posts/{id}` — 删除（作者本人 或 P0）
- `GET /api/blog/posts/{id}/comments` — 评论列表（公开）
- `POST /api/blog/posts/{id}/comments` — 评论（需登录）
- `POST /api/blog/posts/{id}/like` — 点赞（需登录）
- `GET /api/blog/moderation/pending` — 待审核列表（P0）
- `POST /api/blog/moderation/{id}/approve` — 通过审核（P0）
- `POST /api/blog/moderation/{id}/reject` — 拒绝审核（P0）
- `POST /api/blog/reports` — 举报（需登录）

**数据模型**：
```
blog_posts: id, author_id, title, slug, content, status, quality_score, views, created_at
blog_comments: id, post_id, author_id, content, parent_id, created_at
blog_likes: id, post_id, user_id, created_at
blog_reports: id, post_id, reporter_id, reason, created_at
```

**审核流程**：
```
发帖 → status=pending → 敏感词过滤
    → 通过 → status=under_review（待 P0 抽审）
    → 不通过 → status=rejected，通知作者
    → P0 抽审通过 → status=published
    → 举报触发 → status=throttled（降流）→ P0 审核 → published 或 banned
```

### 2.3 oss 插件

**路由**：
- `POST /api/oss/upload` — 用户上传文件
- `GET /api/oss/files/{file_id}` — 下载文件
- `DELETE /api/oss/files/{file_id}` — 删除文件（作者本人 或 P0）
- `GET /api/oss/my` — 我的文件列表
- `POST /api/oss/external/{tenant_id}/upload` — 外部写入
- `GET /api/oss/external/{tenant_id}/files` — 外部文件列表
- `GET /api/oss/storage/stats` — 存储统计

**数据模型**：
```
oss_files: id, owner_id, path, size, mime_type, storage_type, created_at, last_accessed
```

**存储路径**：
```
/storage/
├── users/{user_id}/         # 用户上传
├── p1/{user_id}/            # P1 独立空间
├── external/{tenant_id}/    # 外部写入
└── private/                 # 个人私有文件（冷热分层）
```

**服务**：
- `StorageService` — 本地文件操作、路径安全检查、冷热分层调度
- `CloudStorageService` — 阿里云 OSS 操作（上传/下载/列举）

### 2.4 crawler 插件

**数据模型**：
```
crawler_seeds: id, url, source, depth, discovered_at, status
crawler_tasks: id, url, status, priority, created_at, completed_at
crawler_results: id, task_id, url, title, content_hash, cleaned_content, created_at
crawler_links: id, source_result_id, url, status, discovered_at
```

**调度流程**：
```
预置种子 URL → discoverer 扫描 → 提取链接 → 存入 crawler_links
    → fetcher 轻量并行请求（能拿则拿，拿不到则跳过）→ cleaner 清洗 → 存 crawler_results
    → 结果中的链接 → 有新链接 → 升级为种子 → 回到 discoverer 队列
```

**并发控制**：
- 同站请求间隔 ≥ N 秒（可配置）
- 总并发数上限（可配置）
- 遵守 robots.txt

### 2.5 github_proxy 插件

**路由**：
- `GET /api/gh/repos/{owner}/{repo}` — 仓库信息
- `GET /api/gh/repos/{owner}/{repo}/contents/{path}` — 文件内容
- `GET /api/gh/repos/{owner}/{repo}/issues` — Issue 列表
- `GET /api/gh/repos/{owner}/{repo}/pulls` — PR 列表
- `...` — 完整 GitHub API 映射

**前端**：
- 完整 GitHub 风格界面，Vue 组件

**中转方案**：待调研确定（见 docs/05-GitHub中转方案调研.md）

### 2.6 cloud_integration 插件

**路由**：
- `POST /api/cloud/tasks` — 创建训练任务
- `GET /api/cloud/tasks` — 任务列表
- `GET /api/cloud/tasks/{id}` — 任务详情
- `POST /api/cloud/tasks/{id}/cancel` — 取消任务
- `GET /api/cloud/tasks/{id}/metrics` — GPU 指标时序
- `GET /api/cloud/tasks/{id}/logs` — 训练日志
- `GET /api/cloud/costs` — 费用汇总

**数据模型**：
```
cloud_tasks: id, provider, instance_type, instance_id, status, cost, created_at, completed_at
cloud_metrics: id, task_id, gpu_util, mem_used, mem_total, temperature, loss, timestamp
```

## 4. 数据库表详细设计

### 4.1 用户表

```sql
CREATE TABLE users (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username    VARCHAR(64) UNIQUE NOT NULL,
    password_hash VARCHAR(256) NOT NULL,
    level       INTEGER NOT NULL DEFAULT 5,           -- P0~P5
    blog_quality_level INTEGER NOT NULL DEFAULT 0,     -- 博客质量等级
    created_at  TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at  TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

### 4.2 博客表

```sql
CREATE TABLE blog_posts (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    author_id   UUID REFERENCES users(id),
    title       VARCHAR(256) NOT NULL,
    slug        VARCHAR(256) UNIQUE NOT NULL,
    content     TEXT NOT NULL,
    status      VARCHAR(32) NOT NULL DEFAULT 'pending',
    quality_score INTEGER NOT NULL DEFAULT 0,
    views       INTEGER NOT NULL DEFAULT 0,
    created_at  TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE TABLE blog_comments (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    post_id     UUID REFERENCES blog_posts(id),
    author_id   UUID REFERENCES users(id),
    content     TEXT NOT NULL,
    parent_id   UUID REFERENCES blog_comments(id),
    created_at  TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE TABLE blog_likes (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    post_id     UUID REFERENCES blog_posts(id),
    user_id     UUID REFERENCES users(id),
    created_at  TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    UNIQUE(post_id, user_id)
);

CREATE TABLE blog_reports (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    post_id     UUID REFERENCES blog_posts(id),
    reporter_id UUID REFERENCES users(id),
    reason      TEXT,
    created_at  TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

### 4.3 文件表

```sql
CREATE TABLE oss_files (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    owner_id    UUID REFERENCES users(id),  -- NULL = 个人私有文件
    tenant_id   VARCHAR(64),                -- 外部租户
    path        VARCHAR(1024) NOT NULL,
    size        BIGINT NOT NULL,
    mime_type   VARCHAR(128),
    storage_type VARCHAR(32) NOT NULL DEFAULT 'local',  -- local / aliyun
    is_private  BOOLEAN NOT NULL DEFAULT FALSE,
    created_at  TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    last_accessed TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

### 4.4 爬虫表

```sql
CREATE TABLE crawler_seeds (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    url         TEXT NOT NULL,
    source      VARCHAR(64),
    depth       INTEGER NOT NULL DEFAULT 0,
    status      VARCHAR(32) NOT NULL DEFAULT 'pending',
    discovered_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE TABLE crawler_tasks (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    url         TEXT NOT NULL,
    status      VARCHAR(32) NOT NULL DEFAULT 'pending',
    priority    INTEGER NOT NULL DEFAULT 0,
    created_at  TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    completed_at TIMESTAMP WITH TIME ZONE
);

CREATE TABLE crawler_results (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    task_id     UUID REFERENCES crawler_tasks(id),
    url         TEXT NOT NULL,
    title       VARCHAR(512),
    content_hash VARCHAR(64),
    cleaned_content TEXT,
    created_at  TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

### 4.5 云训练表

```sql
CREATE TABLE cloud_tasks (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    provider    VARCHAR(32) NOT NULL,
    instance_type VARCHAR(32),
    instance_id VARCHAR(128),
    status      VARCHAR(32) NOT NULL DEFAULT 'pending',
    cost        DECIMAL(10, 2) DEFAULT 0,
    created_at  TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    completed_at TIMESTAMP WITH TIME ZONE
);

CREATE TABLE cloud_metrics (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    task_id     UUID REFERENCES cloud_tasks(id),
    gpu_util    FLOAT,
    mem_used    BIGINT,
    mem_total   BIGINT,
    temperature FLOAT,
    loss        FLOAT,
    timestamp   TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

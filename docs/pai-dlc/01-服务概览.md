# 01 — PAI-DLC 服务概览

## 什么是 PAI-DLC？

PAI-DLC（Deep Learning Containers）是阿里云人工智能平台 PAI 提供的**云原生深度学习训练平台**。

核心能力：将训练任务以容器方式提交到 Kubernetes 集群执行，支持单机单卡、单机多卡、多机多卡等多种训练模式。

## 核心概念

### 1. 工作空间（Workspace）
PAI 的顶层组织单元。所有训练任务、数据集、代码配置都归属到某个工作空间。
- 新用户首次使用需要创建**默认工作空间**
- 一个账号可以有多个工作空间

### 2. 计算资源组
DLC 训练任务运行在哪种资源池上：

| 类型 | 说明 | 计费 |
|------|------|------|
| **公共资源组** | PAI 托管的共享 GPU 资源池，Serverless | 按任务运行时长计费 |
| **专有资源组** | 用户自己的 ECS GPU 实例组成的资源池 | DLC 平台费免费，付 ECS 费用 |

### 3. 训练任务（Job）
一次训练运行就是一个 Job。主要属性：
- **JobType**：PyTorchJob / TensorFlowJob / RayJob / SlurmJob
- **DisplayName**：任务名称
- **Status**：Pending / Running / Succeeded / Failed / Stopped
- **ResourceConfig**：GPU 规格、节点数
- **CodeSource**：代码来源
- **DataSources**：挂载的数据源

### 4. 代码配置（CodeSource）
训练代码的获取方式：
- Git 仓库（GitHub、GitLab、阿里云 Codeup 等）
- 本地上传打包
- 提前创建的代码源 ID

### 5. 数据集（DataSource）
训练数据的挂载来源：
- OSS（对象存储）
- NAS（文件存储）
- CPFS（并行文件系统）
- MaxCompute

## 产品架构

```
┌─────────────────────────────────────────────┐
│              PAI 控制台 / SDK / API          │
│        (提交训练任务的方式)                    │
├─────────────────────────────────────────────┤
│               PAI-DLC 训练平台               │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  │
│  │ PyTorch  │  │TensorFlow│  │   Ray    │  │
│  │  Job     │  │   Job    │  │   Job    │  │
│  └──────────┘  └──────────┘  └──────────┘  │
│           ↓ Kubernetes 调度 ↓               │
├─────────────────────────────────────────────┤
│              计算资源层                       │
│  ┌──────────────┐  ┌──────────────────┐    │
│  │ 公共资源组    │  │  专有资源组       │    │
│  │ (Serverless) │  │ (ECS GPU 实例)   │    │
│  └──────────────┘  └──────────────────┘    │
├─────────────────────────────────────────────┤
│              存储层                          │
│  OSS  │  NAS  │  CPFS  │  MaxCompute       │
└─────────────────────────────────────────────┘
```

## 与其他 PAI 子产品的关系

| 产品 | 用途 | 与 DLC 的关系 |
|------|------|-------------|
| **PAI-DSW** | 交互式建模（类似 Jupyter Notebook） | 在 DSW 里写代码，提交到 DLC 跑训练 |
| **PAI-DLC** | 分布式训练 | 本文主角 |
| **PAI-EAS** | 模型在线部署（推理服务） | DLC 训练完的模型部署到 EAS |
| **PAI-Designer** | 可视化建模（拖拽式流水线） | 可用 Designer 的 DLC 节点提交任务 |
| **PAI-QuickStart** | 一键部署开源模型 | 内部也用 DLC 拉起训练 |

## 完整训练流程

```
1. 准备数据 → 上传 OSS/NAS → 创建数据集
2. 准备代码 → 提交 Git / 本地打包 → 创建代码配置
3. 创建训练任务 → 选镜像 / 选资源 / 写命令 → 提交
4. 监控训练 → 查看日志 / TensorBoard
5. 获取产物 → 模型输出保存到 OSS/NAS
```

## 支持的地域

PAI-DLC 在以下地域可用（部分）：
- 华东1（杭州）`cn-hangzhou`
- 华东2（上海）`cn-shanghai`
- 华北2（北京）`cn-beijing`
- 华北6（乌兰察布）`cn-wulanchabu`（价格较低）

不同地域的 GPU 资源池和价格不同。

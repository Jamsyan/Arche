# 02 — 快速入门：提交第一个训练任务

## 前置准备

1. **开通 PAI 服务**：登录阿里云控制台 → 搜索 "PAI" → 开通
2. **完成授权**：首次使用 DLC 需要完成 RAM 角色授权，系统会自动创建
3. **创建默认工作空间**：在 PAI 控制台首页操作

## 方式一：通过控制台创建（推荐新手）

### 步骤 1：进入 DLC 页面
登录 PAI 控制台 → 选择工作空间 → 左侧导航栏选择 **"分布式训练（DLC）"**

### 步骤 2：创建训练任务
点击 **"创建训练任务"**，填写以下信息：

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| 任务名称 | `mindcore-test` | 自定义 |
| 资源组 | `公共资源组` | 试用阶段用公共的 |
| 框架类型 | `PyTorch` | 我们用的框架 |
| 镜像 | 选择 PyTorch 官方镜像 | 见 [03-镜像系统](03-镜像系统.md) |
| 资源规格 | 根据可用资源选择 | 例如 1卡 A10 |

### 步骤 3：挂载数据
- 在 **"数据集"** 选项卡中选择已创建的 OSS 或 NAS 数据集
- 设置挂载路径，如 `/mnt/data`

### 步骤 4：配置启动命令
在 **"启动命令"** 中填入：
```bash
cd /code && python train.py
```

### 步骤 5：提交
点击提交，任务会进入 Pending → Running 状态。

## 方式二：通过 Python SDK 提交

### 安装
```bash
pip install pai
```

### 配置
```bash
python -m pai.toolkit.config
```
按提示输入 AccessKey、工作空间 ID、OSS Bucket 等。

### 代码示例
```python
from pai.estimator import Estimator

estimator = Estimator(
    image_uri="registry.cn-hangzhou.aliyuncs.com/pai-dlc/pai-pytorch-training:2.0-gpu-py310-cu118",
    source_dir="./src",              # 本地代码目录
    command="python -m mindcore train --config config/demo_train.yaml",
    instance_type="ecs.gn7i-c8g1.2xlarge",  # A10 GPU
    instance_count=1,
    max_run_time=7200,              # 2 小时
    outputs=["/code/checkpoints/"],  # 输出目录
)

estimator.fit(
    inputs={
        "data": "oss://your-bucket/dataset/",
    },
    job_name="mindcore-pytorch-job",
)
```

## 方式三：通过 DLC 命令行工具

```bash
# 提交 PyTorch 任务
dlc submit pytorchjob --job_file ./pytorch.params
```

`pytorch.params` 文件内容：
```ini
name=mindcore-job
image=registry.cn-hangzhou.aliyuncs.com/pai-dlc/pai-pytorch-training:2.0-gpu-py310-cu118
command=python train.py
worker_count=1
ecs_spec=ecs.gn7i-c8g1.2xlarge
code_source=/path/to/code
data_sources=oss://your-bucket/dataset/:/mnt/data
```

## 监控与日志

- **控制台**：在 DLC 任务列表点击任务名称 → 查看日志
- **SDK 查询**：
  ```python
  from pai.dlc import Job
  job = Job.get(job_name="mindcore-pytorch-job")
  print(job.get_logs())
  ```
- **TensorBoard**：DLC 支持 CreateTensorboard 接口

## 常见问题

### Q：任务一直处于 Pending？
公共资源组 GPU 资源可能紧缺，换一个地域（如乌兰察布）或等低峰期提交。

### Q：如何保存训练产物？
将 checkpoint 写到挂载的 OSS 或 NAS 路径下，任务结束后自动保存。

### Q：训练中途失败怎么办？
DLC 支持 AIMaster 容错引擎，可自动重启失败的 Pod。也可在 SDK 中配置重试策略。

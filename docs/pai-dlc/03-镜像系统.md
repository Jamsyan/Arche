# 03 — 镜像系统

## 官方预置镜像

PAI 提供了深度学习训练框架的预置镜像，开箱即用。

### 镜像地址格式

```
registry.<region>.aliyuncs.com/pai-dlc/pai-pytorch-training:<tag>
```

- `<region>`：地域，如 `cn-hangzhou`、`cn-shanghai`
- VPC 内网地址（推荐）：`registry-vpc.<region>.aliyuncs.com/pai-dlc/...`

### PyTorch 训练镜像

常见的 PyTorch 官方训练镜像 tag 格式：

| Tag 示例 | 说明 | 适用场景 |
|---------|------|---------|
| `torch-1.12-cuda11.3-py38-acc-230219` | PyTorch 1.12 + CUDA 11.3 + Python 3.8 | 老项目兼容 |
| `1.7.1-gpu-py37-cu110-ubuntu18.04` | PyTorch 1.7.1 + CUDA 11.0 | 兼容性测试 |
| `2.0-ubuntu20.04-py3.10-cuda11.8-megatron-patch-llm` | PyTorch 2.0 + CUDA 11.8 + LLM 训练 | **大模型训练** |
| `2.1-gpu-py310-cu121` | PyTorch 2.1 + CUDA 12.1 | 较新版本 |

## MindCore 推荐镜像

基于 MindCore 项目需求（PyTorch + PyTorch Lightning + Python 3.13）：

### 推荐选项
```
registry.cn-hangzhou.aliyuncs.com/pai-dlc/pai-pytorch-training:2.0-gpu-py310-cuda11.8
```

### 理由
- PyTorch 2.0 支持 Flash Attention、compiled training
- CUDA 11.8 兼容 A10/V100/T4 等主流 GPU
- Python 3.10 稳定且兼容性好

### 注意事项
- MindCore 本地使用 Python 3.13，但 DLC 镜像中通常是 3.8-3.10
- 需要在自定义 Dockerfile 中安装额外依赖（pytorch-lightning、rich 等）

## 自定义镜像构建

如果官方镜像不够用，可以基于官方镜像构建自定义镜像：

### Dockerfile 示例
```dockerfile
FROM registry.cn-hangzhou.aliyuncs.com/pai-dlc/pai-pytorch-training:2.0-gpu-py310-cuda11.8

# 安装额外依赖
RUN pip install --no-cache-dir \
    pytorch-lightning \
    rich \
    tensorboard \
    pyyaml

# 设置工作目录
WORKDIR /code

# 复制项目代码（可选，也可通过 CodeSource 挂载）
COPY . /code/
```

### 推送到阿里云 ACR
```bash
# 登录
docker login --username=your-username registry.cn-hangzhou.aliyuncs.com

# 构建
docker build -t registry.cn-hangzhou.aliyuncs.com/your-namespace/mindcore-training:latest .

# 推送
docker push registry.cn-hangzhou.aliyuncs.com/your-namespace/mindcore-training:latest
```

### 在 DLC 中使用
在创建训练任务时，指定镜像 URI 为你的自定义镜像地址即可。

## 镜像选择建议

| 场景 | 推荐方案 |
|------|---------|
| 快速试训 | 直接用官方 PyTorch 镜像，pip install 依赖 |
| 依赖复杂 | 构建自定义镜像，预装所有依赖 |
| 生产训练 | 自定义镜像 + 锁定版本，保证可复现 |
| 省钱 | 使用较小的官方基础镜像，最小化启动时间 |

# 05 — 存储与挂载

## 存储类型对比

| 存储类型 | 特点 | 适合场景 | 性能 |
|---------|------|---------|------|
| **OSS** | 对象存储，HTTP 访问 | 数据集存储、训练产物 | 中等（ossfs 2.0 优化后可达 10GB/s） |
| **NAS** | 文件存储，POSIX 兼容 | 频繁读写的小文件 | 好 |
| **CPFS** | 并行文件系统 | 大规模分布式训练 | 最好 |
| **MaxCompute** | 大数据分析平台 | 表格类数据 | 特定场景 |

## OSS 挂载

### 方式 1：ossfs 2.0 挂载（推荐）

DLC 2025 年新增 ossfs 2.0 挂载方式，性能大幅提升。

在 CreateJob API 的 `Options` 参数中设置：
```json
{"mountType": "ossfs"}
```

### 方式 2：通过数据集挂载

1. **在 PAI 控制台创建数据集**：
   - 进入工作空间 → AI资产管理 → 数据集
   - 选择数据源类型为 OSS
   - 填入 OSS 路径：`oss://your-bucket/dataset/`
   - 设置挂载路径：`/mnt/data`

2. **在训练任务中引用**：
   - 创建训练任务时在"数据集"选项卡选择该数据集

### 方式 3：SDK 直接读写（非挂载）

在训练代码中直接使用 oss2 SDK 读写 OSS：
```python
import oss2

auth = oss2.Auth(access_key_id, access_key_secret)
bucket = oss2.Bucket(auth, 'https://oss-cn-hangzhou.aliyuncs.com', 'your-bucket')

# 下载
bucket.get_object_to_file('dataset/cleaned_00000.jsonl', '/tmp/data.jsonl')

# 上传
bucket.put_object_from_file('results/model.pt', '/tmp/model.pt')
```

### OSS 路径格式
```
oss://<bucket-name>/<path>/
```
例如：`oss://mindcore-data/dataset/v1/cleaned_00000.jsonl`

## NAS 挂载

### 前置条件
1. 在 NAS 控制台创建文件系统（通用型 NAS 或极速型 NAS）
2. 创建挂载点（确保与 DLC 在同一 VPC）
3. 记录 FileSystemId

### 通过 CreateJob API 挂载

```python
from alibabacloud_pai_dlc20201203.models import (
    CreateJobRequest,
    DataSourceItem,
)

data_source = DataSourceItem(
    data_source_type="NAS",
    file_system_id="your-nas-fs-id",
    mount_path="/mnt/nas",
    access_permission="RW",  # RW=读写, RO=只读
)

request = CreateJobRequest(
    display_name="my-job",
    data_sources=[data_source],
    # ... 其他参数
)
```

### 通用型 NAS 性能优化

通过高级配置启用 nconnect：
```json
{"nconnect": 8}
```

## CPFS 挂载

CPFS 适合大规模分布式训练场景。

```python
data_source = DataSourceItem(
    data_source_type="CPFS",
    file_system_id="your-cpfs-fs-id",
    mount_path="/mnt/cpfs",
    access_permission="RW",
)
```

## 代码源配置（CodeSource）

训练代码可以通过以下方式提供：

### 方式 1：Git 仓库

```python
from alibabacloud_pai_dlc20201203.models import CodeSourceItem

code_source = CodeSourceItem(
    code_repo_type="git",
    code_url="https://github.com/your-org/mindcore.git",
    revision="main",        # 分支或 commit
    code_dir="/code",       # 容器内挂载路径
)
```

### 方式 2：提前创建代码源

先在控制台或 SDK 创建代码源，然后在 Job 中引用：
```python
request = CreateJobRequest(
    code_source_id="codesource-xxxxx",
    # ...
)
```

### 方式 3：本地打包上传

将代码打成 tar.gz 上传到 OSS，然后在启动命令中解压：
```bash
cd /code && curl -sSf http://oss-url/code.tar.gz | tar xz && python train.py
```

## 存储读写权限

| DataSourceType | AccessPermission | 说明 |
|---------------|-----------------|------|
| OSS | RO | 只读挂载，训练数据推荐 |
| OSS | RW | 读写挂载，训练产物输出 |
| NAS | RO | 只读 |
| NAS | RW | 读写，checkpoint 保存 |

## MindCore 推荐存储方案

```
OSS: oss://mindcore-data/
├── dataset/              # 训练数据
│   └── v1/
│       └── cleaned_00000.jsonl
├── code/                 # 训练代码（可选）
│   └── mindcore.tar.gz
└── results/              # 训练产物
    └── pretrain_demo/
        ├── checkpoints/
        ├── report/
        └── config.yaml

挂载配置:
- OSS → /mnt/data (RO)    # 训练数据只读
- OSS → /mnt/output (RW)  # 训练产物可写
```

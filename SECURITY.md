# 安全政策

## 报告安全漏洞

如果你发现了一个安全漏洞，请通过以下方式报告：

- 在 GitHub 创建 Issue，标题标注 `[SECURITY]`
- 直接联系项目维护者

**请不要公开披露未修复的安全漏洞**，以便有时间进行修复。

## 支持版本

| 版本 | 是否支持安全更新 |
|------|-----------------|
| 0.1.x (当前) | 是 |
| < 0.1 (历史) | 否 |

## 安全实践

本项目遵循以下安全实践：

- **敏感信息**：所有密钥、密码、Token 等通过环境变量管理，不提交到代码库
- **依赖更新**：定期更新依赖以修复已知安全漏洞
- **代码审查**：Pull Request 必须通过代码审查后方可合并

# Quick Commands — 快捷指令任务注册表

本 Skill **本身即是一个总斜杠命令**。你不需要记忆 `/{某个具体命令}`，只需用**自然语言描述你要执行的任务**。

## 工作方式

1. 你说出要执行的任务，例如：
   - "修复这个函数"
   - "解释这段代码的逻辑"
   - "为这个模块生成测试"
   - "重构一下这个组件"
2. AI 读取你的请求，在下方**任务注册表**中查找匹配的任务
3. **匹配成功** → 按任务模板执行并输出
4. **无匹配** → 礼貌告知"当前注册表中没有对应的任务"，并列出可用任务供参考

## 任务注册表

| 触发关键词 | 说明 |
|-----------|------|
| 修复 / bug / 问题 | 修复选中代码的 bug |
| 解释 / 说明 / 它在做什么 | 中文解释代码功能和逻辑 |
| 测试 / 单测 / 单元测试 | 为选中代码生成单元测试 |
| 重构 / 重写 / 优化结构 | 重构代码，保持行为不变 |
| 文档 / 注释 / doc | 为代码生成文档注释 |
| 性能 / 优化 / 慢 | 优化代码性能 |
| 审查 / review / 检查质量 | 全面审查代码质量 |

## 各任务模板

### 修复代码

分析用户指出的代码，找出 bug 并修复。输出格式：

```
## 🐛 Bug 分析
- **问题**: ...
- **原因**: ...

## 🔧 修复方案
- (只输出需要改动的代码，不要改动无关行)
```

### 解释代码

用中文解释代码的功能、输入输出和核心逻辑。初学者也能看懂。

```
## 📖 功能概述
...

## 🔄 执行流程
...

## 📝 关键细节
...
```

### 生成测试

为用户选中的代码生成 pytest 单元测试（后端）或 vitest 测试（前端）。遵循项目已有的测试风格。

- 覆盖正常路径、边界条件和异常情况
- 使用项目已有的 fixture 和 conftest
- 不要 mock 无关的内容

### 重构代码

遵循项目的编码规范重构代码。要求：
- 保持行为完全不变
- 去除重复代码，提取公共逻辑
- 遵循项目的命名规范和架构约定

输出格式：重构前后的关键对比。

### 生成文档注释

为代码生成文档注释（项目使用中文注释）。
- 函数/方法：说明功能、参数、返回值
- 类：说明职责、用法
- 模块：说明整体作用

### 优化性能

分析代码的性能瓶颈，提出优化方案并产出优化后的代码。

```
## ⚡ 性能分析
- 瓶颈: ...
- 原因: ...

## 🚀 优化方案
...

## 📊 预期效果
...
```

### 代码审查

对代码进行全面的代码审查。按以下维度评估：

```
## 📋 代码审查报告

### ✅ 正确性
- 逻辑是否正确
- 边界条件是否处理

### 🎨 代码质量
- 命名、结构、可读性
- 是否遵循项目规范

### ⚠️ 潜在问题
- 性能隐患
- 安全风险
- 可维护性问题

### 💡 改进建议
...
```

## 如何添加新任务

在 **任务注册表** 表中新增一行（填写触发关键词和说明），然后在下方添加对应的 `### 任务名` 模板即可。

```markdown
| 格式化 / 排版 / lint | 格式化选中代码 |
```

```markdown
### 格式化代码

使用 ruff format（Python）或 prettier（前端）格式化 `{args}` 中的代码...
```

## 拒绝规则

当用户请求明显不在任务注册表范围内时（例如"帮我写一个电商网站"、"分析股票数据"），回复：

> 当前快捷指令任务注册表中没有匹配「{用户请求}」的任务。可用任务：修复、解释、测试、重构、文档、优化性能、代码审查。如需新增任务，请编辑 `.trae/skills/quick-cmds/SKILL.md` 的任务注册表。
